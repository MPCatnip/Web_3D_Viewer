Look & Feel Style Guide

A flat, light, paper-white interface built from rounded white panels, 2 px borders, and pill-shaped controls, all driven by a single mint-green accent. Copy the token block, then lift the component rules you need — every example below ships with its exact CSS.

01Principles

Six rules hold the whole system together. When in doubt, follow these rather than inventing a new pattern.

Flat & light

No gradients on surfaces, no heavy chrome. A near-white background (--bg) with pure-white panels. Depth comes from soft shadows, never from borders alone.

2 px borders

Almost every control and panel has a 2 px solid border in --border. It thickens or shifts to an accent tint on hover/active — the border is the primary state signal.

Rounded, pill-friendly

Panels use 14px, controls 10–12px, and toggles/chips/floating bars go fully round (999px). Soft corners everywhere.

One accent, used sparingly

Mint green (--accent) means "active / selected / focus." Soft tints carry hover and selection washes; the strong accent is reserved for the truly active element.

Quiet typography

System UI font. Headers are tiny, bold, uppercase, letter-spaced. Body text is calm and low-contrast-grey. Numbers use tabular figures.

Calm motion

Transitions are short (.12s) and easing is gentle. Press states nudge translateY(1px). Nothing bounces or distracts from the model.

02Design tokens

The contract of the whole system. Paste this :root block into any stylesheet and the components below just work. To re-theme, edit only the four accent-palette variables at the top — the grey and tint scales recompute via color-mix().

/* Edit these four to re-theme everything */ --accent:#389d86; /* active / focus / links */ --slate:#495965; /* primary buttons, headings */ --gray:#5B6770; /* seed for the grey scale */ --accent-light:#59e8b1; /* active borders, tints */ /* Greys — tracks, hairlines, muted icons */ --g1:color-mix(in srgb, var(--gray) 85%, white); --g2:color-mix(in srgb, var(--gray) 62%, white); --g3:color-mix(in srgb, var(--gray) 35%, white); --g4:color-mix(in srgb, var(--gray) 17%, white); /* Tints — hover washes, selected rows */ --tint-1:color-mix(in srgb, var(--accent-light) 55%, white); --tint-2:color-mix(in srgb, var(--accent-light) 35%, white); --tint-3:color-mix(in srgb, var(--accent-light) 18%, white); --tint-4:color-mix(in srgb, var(--accent-light) 8%, white); /* Surfaces & text */ --bg:#F9FAFA; --panel:#ffffff; --border:#E1E3E7; --border-strong:#C2C7CD; --text:#2b3238; --text-soft:#5B6770; --text-muted:#939DA7; /* Shape & depth */ --radius:14px; --radius-sm:10px; --shadow:0 6px 22px rgba(73,89,101,.12); --shadow-sm:0 2px 8px rgba(73,89,101,.10); --font:'Segoe UI',Roboto,Helvetica,Arial,system-ui,sans-serif; /* Status colors — badges, alerts, meters (each + a soft -bg) */ --success:#2e8b6f; --warning:#b9822b; --danger:#b0413e; --info:#3f6f9c; --success-bg:color-mix(in srgb, var(--success) 10%, white); /* …-bg per color */
Why color-mix()? The grey and tint ladders are derived, not hand-picked. One accent change ripples through every hover wash and track automatically, so the palette never drifts out of sync.

12Headings & prose

Wrap any rich text — a patent abstract, a report section, slide notes — in .prose and headings, paragraphs, links, quotes, and inline code all inherit the system. One class makes raw HTML document-ready.

Heading one

Body copy sits in calm grey at a comfortable measure. Inline links use the accent with a soft underline, and strong text lifts to full contrast. Reference a token like --accent inline.

Heading two

Sections break with generous spacing. Pull a key point out as a quote:

The claimed apparatus comprises a housing, a sensor array, and a controller configured to actuate the array in response to a detected event.

Heading three

Smaller sub-points carry on here in the same calm tone.

Label-style heading four

A rule cleanly separates passages without needing a heading.
